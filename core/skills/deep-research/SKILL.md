# 深度研究

## 用法

`/deep-research <研究主题>`

面向外部信息的多源研究能力。

## 核心原则

- 每个断言必须引用来源
- 优先近期来源（1年内）
- 交叉验证关键信息
- 零幻觉：不确定的信息标注为不确定

## MCP 工具优先级

| 优先级 | 工具                | 用途                |
| ------ | ------------------- | ------------------- |
| 1      | `mcp__firecrawl__*` | 抓取和解析完整网页  |
| 2      | `mcp__exa__search`  | 语义搜索外部资源    |
| 3      | `read_url_content`  | 无 MCP 时的降级路径 |
| 4      | Web 搜索            | 通用搜索            |

若 `firecrawl` / `exa` MCP 不可用，自动降级到 `read_url_content` + Web 搜索。

## 执行流程

### Step 1: 理解

1. 解析研究主题
2. 识别核心问题和子问题
3. 确定研究范围和深度

### Step 2: 分解

将研究主题拆分为可搜索的子问题（2-5 个）。

### Step 3: 搜索

对每个子问题执行并行搜索：

- 使用最高优先级可用的 MCP 工具
- 每个子问题至少 2 个不同来源
- 记录所有来源 URL

### Step 4: 深读

对搜索结果中的关键页面进行深度阅读：

- 提取核心观点和数据
- 标注来源和日期
- 识别共识和分歧

### Step 5: 综合

交叉验证关键信息，整合输出：

- 已证实的共识（多源一致）
- 有争议的观点（列出各方立场）
- 不确定的信息（明确标注）

### Step 6: 交付

输出研究报告：

```markdown
# 研究报告: [主题]

## 摘要

[2-3 句核心发现]

## 关键发现

### [发现 1]

[内容] [^1]

### [发现 2]

[内容] [^2]

## 建议

[基于研究的行动建议]

## 不确定性与限制

[明确标注未证实的信息]

## 参考来源

[^1]: [URL] - [日期] - [标题]

[^2]: [URL] - [日期] - [标题]
```

## 质量规则

- 每个核心断言至少 1 个来源引用
- 优先使用官方文档和一手资料
- 过期信息（>2年）需标注时间性
- 矛盾信息需列出多方观点
