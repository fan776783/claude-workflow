# 分析与编排输出模板

在 `bug-batch` 的 Phase 3 与 Phase 4 中，需要向用户展示统一的分析结果与修复单元编排结果。此文件提供建议模板，避免把大量示例堆在 `SKILL.md` 中。

## 1. 缺陷分析视图

```markdown
## 缺陷分析视图

| 工单号 | 优先级 | 根因初判 | 模块 | 关系结论 | 建议 |
|--------|--------|----------|------|----------|------|
| p001 | HIGH | 登录态未同步 | auth/store | same_root_cause -> p003 | 合并处理 |
| p002 | 中 | 与 p001 复现路径一致 | auth/store | duplicate_of -> p001 | 归并 |
| p003 | HIGH | token 刷新失效 | auth/api | primary | 建立修复单元 |
```

```markdown
## 缺陷关系矩阵
```
