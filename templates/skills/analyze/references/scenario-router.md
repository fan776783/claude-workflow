# 场景识别指南

根据用户输入识别分析重点，调整双模型的分析侧重。

## 识别规则

| 关键词 | 分析重点 | 模型侧重 |
|--------|----------|----------|
| 上下文、了解、概览 | 项目结构 | 双模型均衡 |
| 在哪、找到、搜索、定位 | 代码定位 | Auggie 为主 |
| 原因、为什么、bug、错误 | 根因分析 | Codex 为主 |
| 性能、慢、优化、瓶颈 | 性能问题 | Codex 后端 + Gemini 前端 |
| 依赖、漏洞、安全、审计 | 安全审计 | Codex 为主 |
| 路由、页面、导航 | 路由结构 | Gemini 为主 |
| 状态、store、redux、pinia | 状态管理 | Gemini 为主 |
| 翻译、i18n、国际化 | 国际化 | Gemini 为主 |
| 需求、拆解、功能点、PRD | 需求分析 | 双模型均衡 |
| 架构、设计、合理、重构 | 架构评审 | Codex 后端 + Gemini 前端 |

## 使用方式

1. 匹配关键词确定分析重点
2. 根据"模型侧重"调整 Prompt 中的分析指令
3. 无法识别时，使用通用双模型并行分析

## 示例

**用户输入**: "性能瓶颈在哪里"

**识别结果**:
- 关键词匹配: "性能"、"瓶颈"
- 分析重点: 性能问题
- 模型侧重: Codex 分析后端性能（N+1、内存泄漏），Gemini 分析前端性能（Bundle 体积、渲染）
