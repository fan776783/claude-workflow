# RESOURCES.md 格式

`RESOURCES.md` 是本主题的可信源精选集。explainer 的知识从这里取,不靠参数化猜测;wisdom 来自这里列出的社区。

## 结构

```md
# {Topic} Resources

## Knowledge

- [Book: _The Science and Practice of Strength Training_ — Zatsiorsky & Kraemer](https://example.com)
  Foundational text on programming and adaptation. Use for: anything to do with periodisation, recovery, intensity zones.
- [Article: "How Much Should I Train?" — Greg Nuckols (Stronger By Science)](https://example.com)
  Evidence-based review of volume landmarks. Use for: weekly set targets per muscle group.

## Wisdom (Communities)

- [r/weightroom](https://reddit.com/r/weightroom)
  High-signal subreddit, moderated against bro-science. Use for: programme critique, plateau troubleshooting.
- Local: Tuesday strength class at {gym name}
  Use for: real-time coaching feedback on lifts.
```

## Rules

- **只收高信任源。** 优先一手来源、公认专家、同行评审、强 moderation 的社区。营销伪装的"教育"不收。
- **每条加注解。** 裸链接三个月后毫无价值。补一行:覆盖什么、什么时候用。
- **按 Knowledge / Wisdom 分组。** 呼应 [SKILL.md](../SKILL.md) 的哲学。一个资源只出现在一组也正常。
- **显式暴露 gap。** mission 需要但找不到好资源的领域,写 `## Gaps` section 列缺口,驱动后续搜索。
- **无情修剪。** 被证明错误、浅薄、偏离 mission 的资源直接删,不留尸体。五个锐利的源好过三十个平庸的。
- **记录社区偏好。** 用户明确不想加入社区时记在这里,后续 session 不再反复提议。
