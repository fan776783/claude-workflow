# Research

"不要重复造轮子"+"不要基于过期知识做决策"的统一入口。

## 用法

```bash
/research <需求 / 研究主题>           # 自动判断走 --internal / --external / 混合
/research --internal <功能需求>        # 仅代码库 + npm/PyPI（原 /search-first）
/research --external <研究主题>        # 仅外部引文（原 /deep-research）
```

## 自动路由

没带参数时按关键词自动选模式：

| 关键词 / 场景 | 模式 |
|---|---|
| "有没有现成 / 工具 / 组件 / MCP / 库"、"先搜后写"、具体功能描述 | `--internal` |
| "最佳实践 / 技术选型 / 对比 / trade-off / 行业"、"深度调研"、概念研究 | `--external` |
| 既要找库又要对比方案 | 混合（先 internal 再 external） |

不确定时先问用户一句，避免跑错方向。

## 核心原则（两种模式共用）

- 不重复造轮子，不基于过期知识做决策
- 每个断言附来源（内部 = 文件 + 行号；外部 = URL + 日期）
- 优先近期来源（外部 1 年内；内部看 `git log` 最近活跃度）
- 交叉验证关键信息，不确定的明确标为不确定

## 模式 A：Internal（代码库 + 生态）

### Step 1 — 需求分析

识别核心能力（输入 / 输出 / 约束），提取关键词，确定搜索范围。

### Step 2 — 并行搜索

| 来源 | 方法 | 重点 |
|------|------|------|
| 代码库 | `mcp__auggie-mcp__codebase-retrieval` | 已有实现 / 可复用组件 |
| npm / PyPI | Web 搜索 | 成熟开源包 |
| GitHub | Web 搜索 | 参考实现 / 最佳实践 |
| MCP 工具 | 检查已安装 MCP server | 可直接调用的工具 |

### Step 3 — 评估每个候选

| 维度 | 问题 |
|------|------|
| 适配度 | 满足核心需求? |
| 维护性 | 最近更新?活跃社区? |
| 依赖代价 | 引入多少新依赖? |
| 集成成本 | 接入需要多少改造? |
| 许可证 | 与项目兼容? |

### Step 4 — 决策

| 决策 | 条件 | 输出 |
|------|------|------|
| **Adopt** | 现有方案满足 > 80% | 安装命令 + 集成示例 |
| **Extend** | 满足 50–80%,可扩展 | 基础包 + 扩展方案 |
| **Build** | 无合适或集成代价过高 | 自建方案 + 参考代码 |

### 与 workflow 的集成

`/workflow-plan` 的 Phase 0 可主动触发本模式,尤其:外部 API 集成 / 数据处理 / 认证授权 / 文件格式解析。

## 模式 B：External（多源引文）

### MCP 优先级

| 级别 | 工具 | 用途 |
|------|------|------|
| 1 | `mcp__firecrawl__*` | 抓取和解析完整网页 |
| 2 | `mcp__exa__search` | 语义搜索外部资源 |
| 3 | `read_url_content` | 无 MCP 时的降级 |
| 4 | Web 搜索 | 通用搜索 |

firecrawl / exa 不可用 → 自动降级到 `read_url_content` + Web 搜索。

### 执行workflow

1. **理解** — 解析主题,识别核心问题 + 子问题,确定范围
2. **分解** — 拆 2-5 个可搜索的子问题
3. **搜索** — 对每个子问题并行搜,每个至少 2 个不同来源,记所有 URL
4. **深读** — 关键页面提取核心观点 / 数据 / 日期,识别共识与分歧
5. **综合** — 交叉验证,分"已证实 / 有争议 / 不确定"三档
6. **交付** — 按下面模板输出

### 报告模板

```markdown
# 研究报告: [主题]

## 摘要
[2-3 句核心发现]

## 关键发现

### [发现 1]
[内容] [^1]

### [发现 2]
[内容] [^2]

## 建议
[基于研究的行动建议]

## 不确定性与限制
[明确未证实的信息]

## 参考来源
[^1]: [URL] - [日期] - [标题]
[^2]: [URL] - [日期] - [标题]
```

### 质量规则

- 每个核心断言至少 1 个来源引用
- 官方文档 / 一手资料优先
- 过期信息（> 2 年）标注时间性
- 矛盾信息列出多方观点
